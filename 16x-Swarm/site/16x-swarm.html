How it works

    
One prompt in. Three agents negotiate. Approved code out.

    

      
You dispatch a task
"Write a Python script to scrape a site and write unit tests." Optionally let ✦ AI refine it first.

      
Planner writes the spec
Objective, deliverables, constraints, and numbered acceptance criteria the QA will enforce.

      
Coder implements
Complete, runnable files — then hands them across the bus for review.

      
QA rejects until it's right
Concrete, numbered issues bounce back to the Coder. The loop repeats until VERDICT: APPROVE.

Documentation

    
Everything you need — also built into the app under Docs.


    

      Quick start
      

        
1. Download and open 16x-Swarm, then hit ▶ Demo — a scripted run shows the full plan → code → reject → fix → approve loop with zero setup.

        
2. For real runs: open Settings, paste an API key from openrouter.ai/keys, and hit Test. The key is encrypted at rest with your OS keychain and only ever sent to OpenRouter.

        
3. Type a coding task and press ⚡ Run Swarm (⌘/Ctrl+Enter). Copy the approved code straight from the Coder terminal.

      

    

    

      Choosing models per agent
      

        
Any OpenRouter model ID works in any slot. Patterns that work well:

        
Adversarial review: anthropic/claude-sonnet-4 as Coder + openai/gpt-4o as QA — cross-vendor review is more critical than self-review.

        
Budget: deepseek/deepseek-chat-v3-0324 everywhere — full runs for pennies.

        
Fast specs: a small model (e.g. openai/gpt-4o-mini) as Planner — specs are short; save the strong model for code.

      

    

    

      AI assistance features
      

        
✦ Refine with AI rewrites a vague prompt into a specific, testable task before dispatch. Better input → tighter spec → fewer rejections → cheaper runs.

        
✦ Explain run (in the Trace Inspector) feeds the entire inter-agent event log to your assist model and returns a plain-language root-cause: why QA rejected, where the loop stalled, and how to fix your prompt or model line-up.

      

    

    

      Debugging with the Trace Inspector
      

        
Open Trace for the full event log: dispatches, hand-offs, verdicts, per-agent latency and token counts. Click any row for raw JSON; Export JSON saves the run.

        
Repeated REJECT on the same issue → the Coder model can't satisfy that criterion; upgrade it or simplify the task. VERDICT: UNKNOWN → the QA model ignored the output contract; pick a more instruction-following model. MAX RETRIES HIT → raise Max iterations or run ✦ Explain.

      

    

    

      Privacy, cost & FAQ
      

        
Where does my data go? Straight from your machine to OpenRouter — there is no 16xBrains backend or telemetry.

        
Cost? A typical three-agent run with two iterations is 5–15k tokens. With budget models that's well under a cent.

        
Languages? Any — say "in Rust" or "as a React component" in your prompt.
