How to use SchemaMind

The five-minute guide. The same docs ship inside the app (Docs tab).

1 · Start a schema, three ways

  • Visual: double-click the canvas → new table. Select it to edit fields, types, PK/unique/required/index flags, defaults, enums and notes in the inspector. Use “Add relation” to link tables — the FK field is auto-created.
  • Script: open the Script tab, type SchemaScript, hit ⌘/Ctrl+Enter.
  • English: in the Script tab’s ✦ bar, type “a clinic system with doctors, patients, appointments and prescriptions” → Generate.

2 · SchemaScript reference

table users {
  id uuid pk
  email string unique !null
  role enum(admin, member) default(member)
  created_at datetime default(now) index
}

ref orders.user_id > users.id          // many-to-one
ref profiles.user_id - users.id        // one-to-one
ref posts.id <> tags.id                // many-to-many

Types: uuid int bigint float decimal string text boolean date datetime time json binary enum · Modifiers: pk unique !null index default(…) note("…") · Defaults now / uuid / autoincrement translate per export target.

3 · Turn on AI

Settings → AI → paste your OpenRouter key and pick a model. Then use ✦ Suggest (inspector), ✦ Review, ✦ Relations and ✦ Docs (toolbar). Your key stays on your machine; schemas are sent only to OpenRouter, only when you click.

4 · Import & export

⌘/Ctrl+I to import SQL, JSON, CSV, Excel or script text — “merge” adds to the current design. ⌘/Ctrl+E to export any of the 10+ formats with live preview, or an Excel data dictionary.

5 · Generate test data

The Data tab creates realistic rows per table (set counts and a seed), respecting every relation in dependency order. Preview in a grid, then export JSON / CSV / Excel / SQL / Mongo.

6 · Projects

One portable .schemamind.json file per project — commit it to git next to your code. ⌘/Ctrl+S save · ⌘/Ctrl+O open · ⌘/Ctrl+Z undo.
