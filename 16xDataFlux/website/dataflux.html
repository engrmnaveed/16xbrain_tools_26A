Why teams migrate — made visible

Database modernization proposals live or die on one question: "why will it be faster?" 16xDataFlux answers it visually, per-table, with a written rationale for every design decision.

🎬

The morph

Child tables physically fly into their parents as embedded arrays. Junction tables dissolve into arrays — or become first-class graph relationships. JOIN arrows vanish as you watch.

📊

Numbers, not vibes

JOINs eliminated (count + %), the worst-case JOIN chain with actual SQL, read amplification per model, unindexed foreign keys, hot tables under contention.

🧠

Deterministic engine

Embed vs reference decisions follow real data-engineering heuristics — ownership, cardinality, growth, shared readers. Same input, same design. No key required.

✨

How to use it

Built-in docs ship inside the app (📖 Docs button) — here's the short version.

1

Load a schema

Pick a preset (e-commerce, healthcare, social, banking), paste CREATE TABLE DDL, open a .sql file — or type a domain like "airline booking system" and let AI generate realistic legacy DDL.

2

Analyze & Morph

The engine parses tables, foreign keys and indexes, then designs both target models with a written rationale per table. Hit the mode switcher: ▦ SQL ⇢ 📄 Document ⇢ ⬡ Graph.

3

Read the analysis

JOIN-elimination stats, the worst JOIN chain in your schema as real SQL, unindexed FKs (with AI-explained production impact + interim fix), hot tables.

4

Plan & scaffold

Strategy writes a 3-phase strangler-fig migration plan. Scripts generates the ETL skeleton. Assistant answers questions grounded in your schema. Export everything.

FAQ

Is it really free?

Yes. It's a showcase of how we think about data modernization at 16xBrains. AI features need your own OpenRouter key (pay-as-you-go, pennies per analysis).

What SQL dialects work?

Standard DDL — MySQL, PostgreSQL, SQLite, MSSQL flavours of CREATE TABLE / CREATE INDEX. It parses structure, not data.

Does my schema get uploaded anywhere?

No. Parsing and transformation are 100% local. Only the optional AI features send schema context to OpenRouter, using your key, when you click them.

Which AI models can I use?

Anything on OpenRouter — Claude, GPT-4o, Gemini, Llama, DeepSeek… We default to Claude 3.5 Sonnet for schema reasoning.

Need the real migration?

16xDataFlux shows the destination. Getting a production system there — zero-downtime, dual-writes, backfill, cutover — is what we do.
