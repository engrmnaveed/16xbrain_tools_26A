Tools / 16x CodeGraph

Free · Open Source · Runs 100% Locally

Refactor large TypeScript codebases
with small, cheap LLMs.

16x CodeGraph is a Mac desktop tool that parses your codebase into an AST-backed dependency graph, then sends your LLM only the function you're refactoring plus its direct dependency signatures. A 9GB local Gemma suddenly performs like a much bigger model — because the app does the reasoning about what code matters.

Why context beats model size

Small models fail at refactoring for one reason: you paste in whole files and they drown. Give them a 60-line function and the exact signatures of the 4 things it calls, and they excel.

your repo (500k LOC) │ ts-morph AST scan → functions, classes, interfaces + who-calls-what ▼ SQLite code graph (~/.codegraph/codegraph.db, incremental re-scans) │ you pick: processPayment() ▼ prompt = target code (60 lines) + 4 dependency signatures ← ~2k tokens, not 200k ▼ Ollama (local gemma2:9b) ─or─ OpenRouter (any cheap model) ▼ streamed diff view → copy the refactored code

Features

01Local code scanner

Point it at any TypeScript project. ts-morph extracts every function, component, class, interface, type, and enum — plus the dependency edges between them.

02Scan once, update fast

The graph lives in a local SQLite database. Files are content-hashed, so re-scans only re-parse what changed and prune what was deleted.

03Targeted selection

Search any entity, see its code, what it depends on and what uses it. Click through the graph like an IDE.

04Provider toggle

One unified LLM service. Flip between local Ollama and OpenRouter's OpenAI-compatible API in Settings — with optional automatic fallback.

05Streaming diff view

Tokens stream in live; when done you get a clean +/- line diff against the original and a one-click copy of the refactored code.

06Private by design

With Ollama, your code never leaves your machine. The graph DB, settings, and prompts are all local.

Setup

Five minutes from clone to first refactor.

1 · Prerequisites

# Node.js ≥ 20  (nodejs.org or via nvm)
node --version

# Rust toolchain (required by Tauri)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# macOS build tools
xcode-select --install

2 · Install & run

git clone https://github.com/16xbrains/codegraph.git
cd codegraph
npm install

# full desktop app (Node engine + Tauri window)
npm run dev

# no Rust installed? run it in the browser instead:
npm run dev:web        # → http://localhost:1420

3 · Pick an LLM

Option A — local & free (recommended):

# install Ollama from ollama.com, then:
ollama pull gemma2:9b
ollama serve           # usually already running

Option B — cloud & cheap: create a key at openrouter.ai, then in the app open ⚙ Settings → OpenRouter and paste the key + a model ID (e.g. google/gemma-2-9b-it). Hit Test connection either way.

Using CodeGraph

Scan your project

Click Scan project and select your repo. A 100k-LOC codebase parses in under a minute; subsequent scans take seconds because unchanged files are skipped by hash.

Find your target

Search by name — processPayment, UserCard, AuthService. Results show kind, file, and export status.

Review the blast radius

Depends on shows what your target calls (these signatures go to the LLM). Used by shows every caller — so you know what to re-test before you paste the result back.

Refactor

Type an instruction — "convert to async/await and add error handling" — and hit Refactor. The prompt meta line shows exactly how small the context was. Review the diff, copy, paste, done.

FAQ

Does my code leave my machine?
Not with Ollama — parsing, the graph database, and inference are all local. With OpenRouter, only the targeted prompt (one entity + dependency signatures) is sent, never whole files or the repo.
Which models work well?
Locally: gemma2:9b, qwen2.5-coder:7b, codellama:13b. On OpenRouter, anything OpenAI-compatible — small code models are usually plenty because the context is so focused.
"Ollama not running" error?
Start it with ollama serve and confirm the endpoint in Settings is http://localhost:11434/api/generate. If the model is missing, run ollama pull <model> — the app will tell you which.
Where is the graph stored?
~/.codegraph/codegraph.db (SQLite, WAL mode). Delete it any time to force a full re-scan.
JavaScript / other languages?
TypeScript (.ts/.tsx) today. The graph schema is language-agnostic, so more parsers can follow.
Does it write changes back to my files?
Not automatically — by design. You review the diff and copy the result. Trust-but-verify is the right posture with small models.
