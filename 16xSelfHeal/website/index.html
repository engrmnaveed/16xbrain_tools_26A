The self-healing story, visualized

Every crash plays out the same sequence your production stack runs — now visible.

Failure

You kill an instance. It stays in the load-balancer pool for a moment — requests hitting it fail and retry against healthy replicas. That brief red flicker is the cost of detection latency, made visible.

Detection

Health checks fail twice in a row → the load balancer ejects the node from its pool. Error rate recovers even though the node is still dead.

Compensation

Traffic redistributes by least-connections. If a dependency is unreachable, circuit breakers open and services serve degraded fallbacks instead of hard failures. DB replicas win leader election and get promoted.

Recovery

The orchestrator boots a replacement (watch it turn blue → green and rejoin the pool). The incident closes and MTTR is recorded in your metrics.

Getting started

Download, install, launch — the e-commerce topology loads with live traffic flowing. Push the big red 💥 INJECT CHAOS button. That's it, you're chaos engineering.

Reading the map

  • Hexagons are running instances; particles are live requests. Bar under each node = load.
  • Green healthy · amber degraded · blue booting · red dead.
  • Hover a node for live stats. Click a node to kill it. Scroll to zoom, drag to pan, double-click to fit.
  • Edge badges: ⛔ circuit breaker open · 🔎 half-open (probing) · ✂️ network partition.

Chaos controls

  • Inject chaos — kill a random healthy instance.
  • Kill DB primary — triggers replica leader election and promotion.
  • Partition / latency / CPU spike — target any group via the selector; effects auto-expire.
  • Traffic ×3 — surge test; overloaded nodes degrade and shed errors.
  • Chaos Monkey — continuous random failures until caged.
  • Scenarios — scripted multi-step incidents (Black Friday, flaky datacenter, cascading failure), or describe your own in the ✨ box and let AI build it.

Topology editing

Scale any group with +/− (watch the orchestrator do the work), add custom service groups with dependencies, or switch presets: 1 Simple Web App · 2 E-commerce · 3 Event Pipeline.

AI setup

Settings → paste an OpenRouter API key → pick a model (or keep openrouter/auto). The key stays on your machine (OS-keychain encrypted where available) and is only sent to openrouter.ai. Everything except AI works offline with no key.

Shortcuts

Space pause · K kill random · M chaos monkey · F fit view · ? in-app guide

Running from source

npm install → npm start · engine test suite: npm test · installers: npm run dist:mac / npm run dist:win. Node 18+, zero runtime dependencies.
